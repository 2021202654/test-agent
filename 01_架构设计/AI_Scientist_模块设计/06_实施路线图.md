# 实施路线图

> 从现有专家系统到 AI Scientist 的渐进式演进
> 4 个阶段，6-8 周完成

---

## Phase 1：单模块验证（第 1-2 周）

### 目标
验证各模块的基础功能，不追求集成

### 任务清单

**HypothesisGenerator**
- [ ] 实现物理约束层（`PhysicsConstraintLayer`）
- [ ] 实现 Gap 识别框架
- [ ] 验证假设生成质量（生成 10 个假设，人工验证）

**ExperimentDesigner**
- [ ] 实现多尺度参数映射（`MultiScaleParameterMapper`）
- [ ] 实现自适应采样策略（`AdaptiveSamplingStrategy`）
- [ ] 生成实验脚本，验证可执行性

**ResultAnalyzer**
- [ ] 实现物理量完整性检验（`PhysicsCompletenessChecker`）
- [ ] 实现跨尺度一致性检验（`CrossScaleConsistencyChecker`）
- [ ] 实现基础统计分析

**PaperWriter**
- [ ] 创建领域专属 LaTeX 模板
- [ ] 实现单章节内容生成
- [ ] 验证图表插入功能

**AutoReviewer**
- [ ] 实现量化评分系统（`QuantitativeReviewer`）
- [ ] 生成评审意见 prompt
- [ ] 验证评分合理性（与人工评分对比）

### 交付物
- 各模块的 Python 实现代码
- 单模块测试报告
- 与人工结果的对比验证

---

## Phase 2：两模块联动（第 3 周）

### 目标
实现相邻模块的数据流转

### 任务清单

**Hypothesis → Experiment**
- [ ] 设计接口协议（JSON Schema）
- [ ] 验证假设到实验方案的转换
- [ ] 端到端测试：输入文献综述 → 输出实验脚本

**Experiment → Result**
- [ ] 实现实验执行与数据采集
- [ ] 验证结果分析器对实验数据的处理

### 交付物
- 模块间接口文档
- 两模块联动的测试报告

---

## Phase 3：闭环测试（第 4 周）

### 目标
全流程跑通：假设 → 实验 → 分析 → 论文 → 评审

### 任务清单

- [ ] 端到端流程编排
- [ ] 每个步骤的输出验证
- [ ] 评审反馈闭环（修改后重新评审）
- [ ] 性能测试（总耗时）

### 交付物
- 完整端到端测试报告
- 生成的论文示例
- 评审报告示例

---

## Phase 4：领域扩展（第 5-8 周）

### 目标
从气固界面催化特性扩展到全气动热耦合

### 任务清单

**第 5-6 周：多物理场集成**
- [ ] 扩展实验设计器支持多物理场耦合
- [ ] 扩展结果分析器支持多物理量分析
- [ ] 更新 LaTeX 模板

**第 7 周：代码复现性增强**
- [ ] 完善代码包生成
- [ ] 添加单元测试
- [ ] 撰写用户手册

**第 8 周：集成优化**
- [ ] 性能优化（并行化、缓存）
- [ ] 用户体验优化（进度提示、错误处理）
- [ ] Gradio UI 集成

### 交付物
- 扩展后的完整系统
- 用户手册
- 示例数据集

## 资源需求

### 硬件资源
- 开发环境：RTX 5060 8GB（已有）
- 计算资源：DSW V100 16GB（已有）

- Python 3.10+
- LLaMA-Factory（已有）
- vLLM（已有）
- Pandoc（已有）
- 新增：matplotlib（可视化）、scipy（统计分析）

### 人力资源
- 核心开发：1 人
- 领域专家咨询：5 小时/周

## 风险与对策

| 风险 | 概率 | 影响 | 对策 |
|------|:----:|:----:|------|
| LLM 生成质量不足 | 中 | 高 | 调整 prompt、增加示例、引入 RAG |
| 物理约束验证复杂 | 中 | 中 | 先简化模型，逐步完善 |
| 模块间接口不兼容 | 低 | 中 | 制定严格的 JSON Schema |
| 计算资源不足 | 低 | 高 | 优化采样策略，减少计算量 |
| 人工验证耗时 | 高 | 中 | 制定简化验证流程 |

## 成功标准

### 技术指标
- 端到端成功率 ≥ 80%
- 生成论文 LaTeX 编译成功率 ≥ 90%
- 评审评分与人工评分相关性 ≥ 0.7

### 科学价值
- 生成至少 1 篇可投稿的论文
- 假设验证率 ≥ 60%
- 跨尺度一致性通过率 ≥ 70%

### 可复现性
- 代码包一键运行成功率 = 100%
- 实验结果复现误差 ≤ 10%
